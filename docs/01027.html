Introduction
Note: 
This new version of the MPFS tool supports the HTTP_NET server which parses the dynamic variables at run time and understands the SSI directives.
The HTTP_NET server also supports persistent connections and TLS and should be used for new projects.
For the obsolete HTTP server use the MPFS2 utility. 

 

The MPFS Utility packages web pages into the MPFS format for efficient storage in an embedded system. It is a graphical application for personal computers that can generate MPFS images (either BIN or C32) for storage in external storage or internal Flash program memory. 

The MPFS utility is also able to interpret Server Side Includes directives like “#include”. The file specified in the “#include” directive won’t be compressed. 

When used to build MPFS images, the MPFS Utility also indexes the dynamic variables found. It uses this information to generate http_net_print.h and http_net_print.c, which ensures that the proper callback functions are invoked as necessary. It also stores this index information along with the file in the MPFS image, which alleviates the task of searching from the embedded device. 

Finally, when developing an application that uses external storage, the MPFS Utility can upload images to the external storage device using the upload functionality built into the HTTP web server or FTP server. The http_net_print.idx file is also generated by the utility, which keeps information of all the dynamic variable details. 

The MPFS utility generates the MPFSSettingDetails.xml file, which contains all of the run-time configured parameters. 

 

Note: 
The new version of the HTTP server, HTTP_NET, parses the dynamic variables at run time. The MPFS utility could still be useful for generating an MPFS image to be stored in the application flash and for getting a list of dynamic variables existent in the web pages. 
Note: 
Web page content between <code> </code> delimiters is ignored by the HTTP server parser.
E.g - <code>~myVariable~</code>, will not expand into a call to the dynamic variable “myVariable”.
