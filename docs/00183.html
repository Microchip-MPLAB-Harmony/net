Running the Demonstration

 

For SAM E70/V71 Xplained Ultra, SAM E54 Xplained Pro Evaluation Kits 

1. Ensure a microSD/SD card is formatted and insert the card microSD/SD card slot on the backside of the hardware board. 

2. Configure the Development Board as given here. For SAM E70 Xplained Ultra make sure jumper J805 has been removed. 

3. Make the demonstration setup as shown here. 

4. Build and download the demonstration project on the target board. 

5. A virtual COM port will be detected on the computer, when the USB cable is connected. 

6. Open a standard terminal application on the computer (like Hyper-terminal or Tera Term) and configure the virtual COM port. 

7. Set the serial baud rate to 115200 baud in the terminal application. 

8. Restart the demonstration by pressing the 'RESET' button on the board. 

9. See that the initialization prints on the serial port terminal. 

10. When the DHCP client is enabled in the demonstration, wait for the DHCP server to assign an IP address for the 

development board. This will be printed on the serial port terminal. 

 

For PIC32MZ Starter Kits 

1. Ensure a microSD/SD card is formatted and insert the card microSD/SD card slot on the backside of the hardware board. 

2. Configure the Development Board as given here. 

3. Make the demonstration setup as shown here. 

4. Build and download the demonstration project on the target board. 

5. A virtual COM port will be detected on the computer, when the USB cable is connected to USB-UART connector. 

6. Open a standard terminal application on the computer (like Hyper-terminal or Tera Term) and configure the virtual COM port. 

7. Set the serial baud rate to 115200 baud in the terminal application. 

8. See that the initialization prints on the serial port terminal. 

9. When the DHCP client is enabled in the demonstration, wait for the DHCP server to assign an IP address for the 

development board. This will be printed on the serial port terminal. 

 

At this moment both the HTTP and the FTP servers should be up and running, waiting for connections. 

For HTTP server details, see HTTP Demonstration 

For FTP server transactions see FTP Server Module
