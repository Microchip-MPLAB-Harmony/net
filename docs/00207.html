Running the Demonstration

There are three sequential commands implemented in this demonstration.

  • setudppacketoptions <hostname> <port> <message> - This command specifies the following parameters for the UDP packet: Destination IP Address or Host name, Destination Port and Message to transfer
  • getudppacketoptions - This command displays the current options
  • sendudppacket - This command sends a UDP packet
 

For SAM E70/V71 Xplained Ultra, SAM E54 Xplained Pro Evaluation Kits

  1. Configure the Development Board as given here. For SAM E70 Xplained Ultra make sure jumper J805 has been removed.
  2. Make the demonstration setup as shown here.
  3. Build and download the demonstration project on the target board.
  4. A virtual COM port will be detected on the computer, when the USB cable is connected.
  5. Open a standard terminal application on the computer (like Hyper-terminal or Tera Term) and configure the virtual COM port.
  6. Set the serial baud rate to 115200 baud in the terminal application.
  7. Restart the demonstration by pressing the 'RESET' button on the board.
  8. See that the initialization prints on the serial port terminal.
  9. When the DHCP client is enabled in the demonstration, wait for the DHCP server to assign an IP address for the development board. This will be printed on the serial port terminal.
 

 

For SAM A5D2 Xplained Ultra Evaluation Kits 

  1. Build the demo by opening the project in the IAR Embedded Workbench
  2. Copy the project output - harmony.bin - on the SD Card that is used to load the executable and run it on the SAM A5D2 board.
    1. The boot loader on the SD card - boot.bin - should be configured to load the harmony.bin image instead of the standard uboot.bin
  3. Configure the Development Board as given here.
  4. Make the demonstration setup as shown here.
  5. Open a standard terminal application on the computer (like Hyper-terminal or Tera Term) and configure the virtual COM port.
  6. Set the serial configuration to: baud rate 115200 baud, 8 data bits, no parity, and 1 stop bit in the terminal application.
  7. Insert the SD Card into J19, SDMMC1, on the SAM A5D2 board and press the reset button. At this moment the board should boot up and display messages at the console.
  8. When the DHCP client is enabled in the demonstration, wait for the DHCP server to assign an IP address for the development board. This will be printed on the serial port terminal
 

 

Set the UDP packet options by typing setudppacketoptions at the terminal console. 

Verify the UDP packet settings by typing getudppacketoptions at the terminal console. 

Send the UDP packet to the destination using the sendudppacket command. 

 

 

After the sendudppacket command is input, the demonstration will make a DNS query to look up the host name and send a UDP packet to that host. 

The output message will be received by the UDP server on the port that is configured by the command setudppacketoptions.
