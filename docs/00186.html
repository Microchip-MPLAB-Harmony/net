Running the Demonstration

 

For SAM E70/V71 Xplained Ultra, SAM E54 Xplained Pro Evaluation Kits

  1. Configure the Development Board as given here. For SAM E70 Xplained Ultra make sure jumper J805 has been removed.
  2. Make the demonstration setup as shown here.
  3. Build and download the demonstration project on the target board.
  4. A virtual COM port will be detected on the computer, when the USB cable is connected.
  5. Open a standard terminal application on the computer (like Hyper-terminal or Tera Term) and configure the virtual COM port.
  6. Set the serial baud rate to 115200 baud in the terminal application.
  7. Restart the demonstration by pressing the 'RESET' button on the board.
  8. See that the initialization prints on the serial port terminal.
  9. When the DHCP client is enabled in the demonstration, wait for the DHCP server to assign an IP address for the development board. This will be printed on the serial port terminal.
 

The demo will use the callback feature of the TCP/IP Stack Initialization. 

It does this in order to use the MAC address that's stored in the on board AT24MAC402 256KByte EEPROM with EUI-48 address chip. 

The application will initiate an I2C transaction with the EEPROM chip. 

When the MAC address is retrieved, the TCPIP_STACK_InitCallback() function will report success and provide the network configuration MAC address 

as part of the TCP/IP configuration data. 

 

After that, the demo behaves like a regular tcpip_tcp_client demo: 

Input the following command from the serial port: openurl <url> - The <url> argument must be a fully formed URL; for instance, http://www.microchip.com/. 

After the command is input, the demonstration will make a DNS query. Then it will open a connection to the requested URL and perform a simple HTTP GET command. 

The response received from the server will be printed on the terminal application through the serial port.
