Introduction

A step-by-step analysis of the DNS Server is as follows:

  1. DNS server can be enabled or disabled at run time.
  2. If the TCP/IP commands module is built in, it contains a "dnss" command that could be used for interfacing with the DNSS module.
  3. A client sends some query to the DNS Server of the TCP/IP stack for a name resolution.
  4. The DNS Server of the TCP/IP Stack first checks the configuration value of replyBoardAddr.
    1. This Boolean flag is passed to the DNSS module at initialization time.
  5. If replyBoardAddr is true, the DNS server will reply to the client query with its own board address.
  7. If replyBoardAddr is false, the DNSS will check its cache for an entry for that name.
    1. If a valid entry is found, the reply is sent to the client requesting the info.
    2. Otherwise no reply is provided.
  8. To update the DNSS cache, new entries can be added either using the DNSS API or directly from the console, using the "dnss" command.
  9. The dnsserv command is used to add, delete and display the DNS server details
  10. Authoritative Name servers and Addition Name servers are not processed.
  11. The client may store the answer to the DNS query in it's own cache for future use.
